### Рекомендації щодо виконання домашніх робіт

Ця гілка призначена для виконання ваших практичних завдань та домашніх робіт. Ви можете використовувати її для створення пулреквестів (Pull Requests) та подальшого злиття з вашою основною гілкою після перевірки.

#### Основні кроки:
1. **Створення та перейменування модулів**:
   - При виконанні нових завдань перейменовуйте номер модуля (наприклад, `module1` на `module2`, `module3` і так далі) відповідно до теми заняття.
   - Це допоможе структурувати ваші роботи та легше орієнтуватися в проекті.

2. **Створення класів**:
   - Створюйте всі необхідні класи в межах пакетів відповідних модулів.
   - Дотримуйтесь стандартів іменування Java (CamelCase) для класів.

3. **Процес Pull Request (PR)**:
   - Після завершення завдання в цій гілці, створіть PR.
   - Це дозволить ментору перевірити ваш код, залишити коментарі та надати поради перед злиттям змін.

Використовуйте цей підхід для кожної наступної теми курсу!
