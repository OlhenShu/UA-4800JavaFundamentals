### Шпаргалка по командам Git

#### Основні команди
- `git init` — ініціалізувати локальний Git-репозиторій
- `git clone <url>` — клонувати віддалений репозиторій

#### Робота зі змінами
- `git status` — перевірити стан робочої директорії та індексу
- `git add <file>` — додати файл до індексу (staging area)
- `git add .` — додати всі змінені файли до індексу
- `git commit -m "повідомлення"` — зафіксувати зміни з коментарем
- `git diff` — переглянути зміни у файлах, що ще не додані до індексу

#### Робота з гілками
- `git branch` — список локальних гілок
- `git branch <name>` — створити нову гілку
- `git checkout <name>` — переключитися на гілку
- `git checkout -b <name>` — створити нову гілку та переключитися на неї
- `git merge <name>` — злити гілку `<name>` у поточну

#### Віддалені репозиторії
- `git remote add origin <url>` — прив'язати локальний репозиторій до віддаленого
- `git push -u origin <branch>` — відправити зміни у віддалений репозиторій та встановити зв'язок
- `git pull` — отримати зміни з віддаленого репозиторію та злити їх
- `git fetch` — завантажити історію з віддаленого репозиторію без злиття

#### Історія та скасування
- `git log` — переглянути історію комітів
- `git reset --hard HEAD` — скасувати всі незафіксовані зміни
- `git revert <commit_id>` — створити новий коміт, що скасовує зміни вказаного коміту
